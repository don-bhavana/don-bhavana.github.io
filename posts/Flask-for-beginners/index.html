Python Flask for Beginners

May 29, 2020

If you are developing a web application in Python, chances are you’re leveraging a framework. This can be Flask, Django, FastAPI or any other similar frameworks. These frameworks are basically a collection of packages or modules which allows developers to write a web app without handling low-level details like protocols, sockets, etc. So in this article we’ll develop a web app with Python Flask using SQLite3 as the database. But first let’s see how it has become so popular.

Why Flask?

Flask is an adaptable, solid, lightweight and future-ready Python based micro web framework. Simply, it’s a framework for building robust APIs. Today, flask works well with a latest architectures like micro-services and server-less which are being followed by technology giants like Netflix, Twitter, Amazon, Walmart, etc. Another reason why flask is flourishing is that it is a non-fuzzy and easy to learn. Easy to learn huh? Now this motivates us to move ahead.

Installing Flask

Guessing that you Pythoneers already have python and pip installed, I’m going to directly jump into installing flask. Open up your terminal(works on all OS) and run the following command.

$ pip install flask

If you want to confirm, start your Python interpreter(type python3 in terminal) and import flask.

>>> import flask

If this statement does not give you any errors, you’re good to go.

Hello World

Let’s kickstart this with a classic ‘Hello World’ program! Now just open up your favourite text editor and start coding.

from flask import Flask

app = Flask(__name__)

@app.route("/")
def helloworld():
    return "HELLO WORLD!!!"

if __name__ == "__main__":
    app.run()

Now I’m gonna save this file as app.py. First navigate to the directory in which this file is saved and run it using the following command (just like any other python file!).

$ python3 app.py

This is what you should be seeing now.

Your application is up and running on the IP 127.0.0.1 (a default IP for localhost) on port 5000. Open a browser and copy 127.0.0.1:5000 into the address bar. And boom - HELLOOO WORLDDDD!!!

This definitely looks like a lot for just Hello world! Let’s now break down the code and try to get an idea about what’s happening. So in
line 1, we are just importing Flask class into our code.

from flask import Flask

Line 2 creates an instance of the class Flask which is imported from the flask package.

app = Flask( __name__)

In the next three lines, firstly we use the route() decorator to tell Flask what URL should trigger our function i.e. binding a function to a URL. app.route(‘/‘) represents the home page.

@app.route('/')
def helloworld(): 
	return "Hello World"

Finally, in the main function, the app.run() will start the flask app server and the server will start listening to the incoming requests.

if __name__ =="__main__":
	app.run()

I guess we can finally make sense out of this code now. So let’s connect to a SQLite database and implement basic GET & POST requests. Without further ado, let’s install SQLite.

Installing SQLite

Installing SQLite in windows:

  • Step 1 − Go to https://www.sqlite.org/download.html, and download the latest 64-bit DLL (x64) for SQLite in the precompiled binaries for Windows section.
  • Step 2 − Create a folder C:\>sqlite and unzip above two zipped files in this folder, which will give you sqlite3.def, sqlite3.dll and sqlite3.exe files.
  • Step 3 − Add C:\>sqlite in your PATH environment variable and finally go to the command prompt and issue sqlite3 command, which opens up the SQLite terminal open up without any errors.

Installing SQLite in macOS and Linux:

Almost all the popular linux distributions and the latest versions of macOS come with pre-installed SQLite. Although if you don’t have it installed, follow the steps below.

  • Step 1 − Go to https://www.sqlite.org/download.html, and download sqlite-autoconf-*.tar.gz from source code section
  • Step 2 − Run the following command
$ tar xvfz sqlite-autoconf-3071502.tar.gz
$ cd sqlite-autoconf-3071502
$ ./configure --prefix=/usr/local
$ make
$ make install

Type ‘sqlite3’ in your terminal , and if you see the sqlite terminal open up with no errors, you’re all set.

Finished? Let’s move to the next step! Next, We need to install the Python library for SQLite.

$ pip install db-sqlite3

So let’s get started with coding? Take a usecase of building Flask APIs with a few endpoints to manage a collection of laptops. These endpoints are GET and POST requests: fetching and creating resources.

First, import flask and sqlite3. In the later part of the tutorial, you’ll understand why we’ve imported jsonify and request classes.

from flask import Flask, jsonify, request
import sqlite3

Connecting the Database

The sqlite.connect() opens a connection to the SQLite database file. If database is opened successfully, it returns a connection object and we are saving that in conn variable.

def get_db():
    conn = sqlite3.connect("laptops.db")
    return conn

SQLite syntax is similar MySQL syntax. So if you have a good grip on SQL, these queries are no brainer.

The conn.cursor() returns a cursor. Cursor is an instance using which you can invoke methods that execute SQLite statements to fetch data from the result sets retuned by the queries.
Next, cursor.execute(table) runs SQL statement to create ‘laptops’ table in SQLite database.

def create_tables():
    conn = get_conn()
    cursor = conn.cursor()
    cursor.execute("""CREATE TABLE IF NOT EXISTS laptops(
                    id INTEGER PRIMARY KEY AUTOINCREMENT,
                    name TEXT NOT NULL,
                    ram integer NOT NULL,
                    processor text NOT NULL
                    )""")

POST and GET Requests

Let’s get into writing that API’s now. We’ll first add data into the database and then view it.

In the route() decorator, we determine the URL slug(/add_laptop) and the method(POST) to be used. POST is the HTTP method that is designed to send data to a server from a client. In the first line of the function, the request.get_json() parses the JSON data received from the client and returns a dictionary (Thus the request class was imported). Later in the execute routine, we give the query and the parameters required. These values sit in the place of ‘?’ in the same sequence. Finally, conn.commit() . This method commits the current transaction. If you don’t call this method, anything you did since the last call to commit() will not be updated to the database.
We are returning the result TRUE confirming the parameters are successfully committed to the database. As we are initiating the communication in JSON(application/json) and hence we must jsonify() the result too. You find this when you configure POSTMAN to test the application.

@app.route("/add_laptop", methods=["POST"])
def insert_laptop():
    laptop_details = request.get_json()
    name = laptop_details["name"]
    ram = laptop_details["ram"]
    processor = laptop_details["processor"]
   # conn = get_db()
    #cursor = conn.cursor()
    statement = "INSERT INTO laptops(name, ram, processor) VALUES (?, ?, ?)"
    cursor.execute(statement, [name, ram, processor])
    conn.commit()
    return jsonify(True)

Next, we’ll write the GET method to fetch and view the data in the database. In the function we execute the query just as we’ve done for the creation of the table, but in this case we have data to be collected. So we use the fetchall() routine which returns a list of the query result records.

@app.route('/list_laptops', methods=["GET"])
def get_laptops():
	#conn = get_db()
    	#cursor = conn.cursor()
    	query = "SELECT id, name, ram, processor FROM laptops"
    	cursor.execute(query)
    	laptops = cursor.fetchall()
    	return jsonify(laptops)

Lastly, in our main function we invoke the create_tables() and app.run(). The run method starts the server at the given host details.

if __name__ == "__main__":
    create_tables()
    app.run(host='127.0.0.1', port=5000)

  • In the mid section, click on body then select raw. You will notice a Dropdown being added at the end of the same pane. Select the JSON type format in it.
  • Now type the data you want to insert into the database in JSON format in the below editor.

And click SEND.

Conclusion

Flask server runs in a single thread mode. It means the application can handle no more than one HTTP request at a time, i.e. the request handling becomes sequential. Whenever you you start a flask app, you’ll see the below message:

WARNING: Do not use the development server in a production environment.
Use a production WSGI server instead.

It means that the default web server will not be able to handle the concurrency, serving static files, etc. very efficiently. In order to solve the problem, we need to have a production-ready web server like Nginx and an application server based on WSGI protocol like uWSGI.
